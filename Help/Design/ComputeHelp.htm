The Compute property is used to compute values and run commands on the device like formulas in Excel. The operations that are currently available are:

=    This assigns the value of one field or a constant to the value of another field. The field name is its internal name in its property sheet. For example, suppose a text field has the internal name "amount" and a picker sub-field has the name "cost". Then "amount = cost" assigns the value of the selected picker "cost" to the "amount" field.  In another example, "amount = 100" assigns the value "100" to the amount field.

+, - , * ,  /    These are arithmetic operations on field values. For example "field1 = field2 + 10" or field1 = field2 * field3". Currently, the use of "(" and ")" are not supported. To check out a good example of using the Compute property, see the template app TapNCharge.

Now()  This will get the current date time and time zone in hours relative to Greenwich mean time. For example "field1 = Now()" After the computation, the value of field1 is "2011-03-26 13:49:23 -0400"

PlayAudio() This will play the audio in a field. For example "PlayAudio(field1)" where field1 is an audio field or hidden field containing an audio URL. This example will play the  audio of the URL in field1.

ShowWebPage() This will show the web page for a URL value of a field. For example "ShowWebPage(url_field)" where url_field is a field or hidden field containing a URL.

The state of a switch or checkbox can be set by assigning the switch or checkbox to either true or false. For example "switch1 = true" will set the field switch1 to "on" if the switch type for switch1 is on-off or set switch1 to "checked" if the switch type is checkbox.

A sequence of statements can be used in one compute field separated by a ";". For example, "field1 = field2; field3 = 100; switch1=false".
