To use your web services with your app , they need to satisfy the following criteria:

  • The Web services must be SOAP based , each with a defined WSDL file.
  • All the input parameters to all Methods need to be of type "string".
  • The response output type must be of type "XmlDocument" . It cannot be a string of XML characters of type "string".
  • One of the response outputs must be the status of the web wervice call that contains a success or failure message.
  • The XML output tags from the web service response that are to be assigned to the mobile phone fields must have unique names in the XML output document. This is because their values are retrieved using global XPath queries. To represent an array of data for a phone field, the XML output must have an array of XML nodes all with the same tag name.

  • Use the following XML syntax examples for outputs:

    •  For single item:

<item>value</item> 

    •  For any 1 dimensional array:

    <array1>aaa</array1>

    <array1>bbb</array1>

    <array1>ccc</array1>

    •  For any 2 dimensional array:

    <array2>

          <dim1>aaa</dim1>

           <dim2>www</dim2>

    </array2>

     <array2>

         <dim1>bbb</dim1>

         <dim2>xxx</dim2>

    </array2>

     <array2>

         <dim1>ccc</dim1>

         <dim2>yyy</dim2>

    </array2>

    •  For any 3 dimensional array:

    <array3>

         <dim1>aaa</dim1>

         <dim2>www</dim2>

         <dim3>mmm</dim3>

    </array3>

     <array3>

         <dim1>bbb</dim1>

         <dim2>xxx</dim2>

         <dim3>nnn</dim3>

    </array3>

     <array3>

         <dim1>ccc</dim1>

         <dim2>yyy</dim2>

         <dim3>ooo</dim3>

    </array3>
